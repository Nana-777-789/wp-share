# 🗺️ ホーム画面・検索（92時間）メンバー別・完全手順書

開発は**「3つのステージ」**に分けて、フロントエンド（画面）とバックエンド（データ）が並行して動けるようにします。

---

## 🚀 ステージ1：【土台作り】デザインの実装 ＆ データの準備
> フロントとバックが、それぞれの得意分野で「素材」を同時に作り始めるフェーズです。

* **🎨 フロントエンド（はやと・ぶんた）のタスク**
  * **作業内容：** 画面のデザイン・表示（25時間）
  * **やること：** HTML/CSSを使って、ホーム画面の見栄え（商品のカード型レイアウトなど）をガリガリ作ります。
  * **ポイント：** まだ本物のデータは届かないので、前につくった `data.json` のような「ダミーのデータ」を使って画面に並べておきます。

* **💻 バックエンド（しおん・なな）のタスク**
  * **作業内容：** WordPressからのデータ連携（25時間）
  * **やること：** WordPressの管理画面を構築し、ゲームのデータ（タイトル、価格、画像など）を登録・出力できるようにします。
  * **ポイント：** フロントチームが後で受け取りやすいように、データをまとめた「共通のデータ出力機能（API）」を裏側で用意しておきます。

---

## 🔄 ステージ2：【合体】検索ロジックと通信の開通
> 2つのチームが作った素材を組み合わせて、システムとして動かします。

* **💻 バックエンド（しおん・なな）のタスク**
  * **作業内容：** 検索や並び替えのロジック（11時間）
  * **やること：** WordPress側で、「キーワード」や「並び替え条件（価格が安い順など）」を受け取って、データを絞り込んで返すプログラム（検索ロジック）を書き上げます。

* **🎨 フロントエンド（はやと・ぶんた）のタスク**
  * **作業内容：** 検索UI ＆ 通信の接続（11時間）
  * **やること：** 画面に「検索窓」や「並び替えボタン」を設置。ボタンが押されたら、バックエンドチームが作ったURLに向けてキーワードを送信する処理を作ります。

> **🤝 全員のDone条件（これができたらステージ2完了！）**
> 画面で「モンハン」と検索したら、バックエンドの検索ロジックが動いて、画面にモンハンだけが表示されること。

---

## 🎨 ステージ3：【仕上げ】リッチコンテンツの連携（20時間）
> 検索が動いたら、最後に画面を豪華にする要素（動画・レビュー）を組み込みます。

* **💻 バックエンド（しおん・なな）のやること**
  * WordPress側に「動画URL」や「星評価（★5など）」の入力欄を追加し、データを出力できるようにします。

* **🎨 フロントエンド（はやと・ぶんた）のやること**
  * 送られてきた動画URLを使って画面にYouTubeプレイヤーを埋め込んだり、星評価を綺麗なアイコン（★★★★☆）で表示するレイアウトを組み込みます。
