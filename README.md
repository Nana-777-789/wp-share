# 🗺️ ホーム画面・検索機能：実装手順書

## 🎪 今回作るもの：ゲームの検索サイト
みんなで作るのは、**「WordPress（商品データを入れる倉庫）」**からゲームのデータを持ってきて、**「ホームページ（お店の棚）」**に綺麗に並べて、お客さんが好きなゲームを検索できるようにする仕組みです。

---

## 🛠️ 全体の手順（3つのステップ）
作業は、この3つの順番で進めると絶対に失敗しません！

### 📦 ステップ1：【素材集め】データと見た目を別々に作る
まずは、バックエンドチームとフロントエンドチームが、それぞれの得意な「素材」を同時に作り始めます。

* **💻 バックエンド（塩・sera）のやること**
  * 「魔法の倉庫（WordPress）」の中に、ゲームの名前や値段（エルデンリング 9240円、など）をポチポチ入力して保存します。
  * 保存したら、「倉庫からデータをいつでも取り出せる準備」をしておきます。
* **🎨 フロントエンド（文）のやること**
  * イラストレーターのように、HTMLとCSSを使って、ホームページの**「見た目のデザイン（お店の棚）」**を綺麗に作ります。
  * ※この時はまだ本物のゲームデータは届かないので、前につくった `data.json`（ニセモノのデータ）を仮の棚に並べておきます。
* **⚙️ フロントエンド（早）のやること**
  * 文さんが作った綺麗な棚に、**「自動で商品を並べるPHPの呪文」**（`foreach` など）を書き加えます。
  * これをしておくと、後で本物のデータが届いたときに、棚が自動でブワッと埋まるようになります！

---

### 🤝 ステップ2：【合体】倉庫とお店の棚をドッキングする！
お互いの素材ができたら、いよいよ2つのチームの成果物を合体させます。

* **💻 バックエンド（塩・sera）のやること**
  * 「倉庫のデータが入っているURL（住所）」をフロントチームに教えてあげます。
* **⚙️ フロントエンド（早・文）のやること**
  * そのURLを自分たちのプログラムに貼り付けて、ニセモノのデータから**「本物の倉庫データ」**に繋ぎ変えます！

> **✨ ステップ2が終わるとできること**
> ホームページを開いたら、WordPressに入力した本物のゲームが自動でズラッと表示されるようになります！

---

### 🔍 ステップ3：【仕上げ】検索ボタンを動かす！
最後に、一番大事な「検索機能」を作ります。

* **💻 バックエンド（塩・sera）のやること**
  * 「『モンハン』って言われたら、倉庫の中からモンハンだけを探し出す」という検索のプログラム（ロジック）をWordPress側に作ります。
* **⚙️ フロントエンド（早・文）のやること**
  * ホームページに「検索窓（文字を入れる箱）」と「検索ボタン」を作ります。
  * ボタンが押されたら、入力された文字をバックエンドチームに送る仕組みを作ります。

---

## 🎯 最終ゴール（これができたら完成！）
> お客さんが検索窓に**「モンハン」**と入れて検索ボタンを押したときに、ホームページの棚に**「モンスターハンターワイルズ」だけがシュッと絞り込まれて表示されれば、全員の任務完了（大成功）**です！
