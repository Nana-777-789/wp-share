## 🚀 開発スタート！最初の環境構築（みんなまずこれをやってね！）
「はい、スタート！」と言われたら、まずは自分のパソコンに開発環境を作ります。以下の手順通りに1つずつ進めてください。

---

### 1️⃣ アプリ「Local」をインストールする
1. パソコンのブラウザで [Localの公式サイト](https://localwp.com/) にアクセスします。
2. ダウンロードという緑色のボタンをクリックします。
3. 画面の指示通りにダウンロードしてください。

---

### 2️⃣ GitHubからコードをダウンロードする
1. GitHubで、wp-shareを開きます。
2. 緑色の **「Code」** ボタンを押し、一番下の **「Download ZIP」** をクリックしてダウンロードします。
3. ダウンロードしたZIPファイルを解凍すると、wp-share-mainの中に **`game_ec`** というフォルダが入っています。

---

### 3️⃣ 「Local」アプリに読み込ませてサイトを起動する
1. インストールしたアプリ **「Local」** を起動します。
2. 左下の **「＋」ボタン（マウスホバーするとAdd local siteと出る）** をクリックします。
3. **「create a new site」** を選択します。
4. さっき解凍して出てきた **`game_ec`** フォルダを丸ごと選択して進めます。
5. 画面の指示通りに進めると、みんなのパソコンの中に自分だけのWordPress環境が自動で作られます！
6. 画面右上にある緑色の **「Start site」** ボタンを押すと、サイトが起動します。
7. 起動したら **「Open site」** ボタンを押すと、ブラウザでテスト用のWordPressサイト（`http://localhost:10003`）が開けるようになります！

---

### 4️⃣ あらかじめ作られた「自分の作業部屋（ブランチ）」に移動する
※これは**開発を始めるときに、最初の1回だけ**やればOKです！

以下のコマンドを打って、自分のチームの部屋に移動してください。

* **しおん・なな チーム（backend）:**

```
git fetch origin
```
```
git checkout backend
```

* **はやと・ぶんた チーム（frontend）:**
```
git fetch origin
```
```
git checkout frontend
```



### 5️⃣ どこにコードを書けばいいの？

各自、VS Codeでダウンロードした **`game_ec`** フォルダを開いて作業します。
たくさんファイルがありますが、**みんなが触るファイルは以下の「2つだけ」**です！

#### 📂 ファイルの場所（階層ルート）

```text
game_ec
└── app
     └── public（WordPressの本体）
          └── wp-content
               └── themes
                    └── 📂 今回みんなで使うテーマのフォルダ
                         ├── 📄 index.php
                         │    └── 🎨【はやと・ぶんた】が編集！
                         │         (テスト用の仮ボタンや、仮の画面を作るファイル)
                         │
                         └── 📄 functions.php
                              └── 💻【しおん・なな】が編集！
                                   (データベースからデータを引っ張るプログラムを書くファイル)

```

## 💻 VS CodeからGitHubへPush（アップロード）する方法

---

### 0️⃣ 【超重要】開発を始める前に：最新の状態にする（pull）
※これは、**「さあ、今日もコードを書くぞ！」という作業開始の直前に必ず毎回**やってください！

他のメンバーが先に進めたコードを自分のパソコンに取り込んで、データを最新状態にします。これを忘れると、後でコードが合体できなくなって大パニックになります！

ターミナルに以下のコマンドを打って Enter を押してください。

* **しおん・なな チーム（backend）：**
```
git pull origin backend
```

* **はやと・ぶんた チーム（frontend）：**
```
git pull origin frontend
```

これで自分のパソコンが最新状態になったので、安心してファイルを作業してOKです！

---

### 1️⃣ 変更したファイルをセーブ対象に選ぶ（add）
今日作業したファイルを、Gitのセーブ対象として登録します。
以下のコマンドを打って Enter を押してください。
```
git add .
```

---

### 2️⃣ セーブを確定して、日記のタイトルをつける（commit）
何を作ったのかがみんなに分かるように、セーブデータに名前をつけます。
以下のコマンドを打って Enter を押してください。
```
git commit -m "コメント"
```
※ "" の中身（メッセージ）は、自分がやった作業内容に合わせて自由に書き換えてね！

---

### 3️⃣ GitHubの自分の部屋へ打ち上げる！（push）
最後につくったセーブデータをGitHubへ送信します。
自分のチームのコマンドを打って Enter を押してください。

* **しおん・なな チーム（backend）：**

```
  git push origin backend
```

* **はやと・ぶんた チーム（frontend）：**

```
  git push origin frontend
```
---

### 🤝 打ち上げた後の仕上げ（マージ申請）
ネットでGitHubの `wp-share` ページを開くと、画面の上に緑色の **「Compare & pull request」** というボタンが出ているので、それをクリックして「〇〇のチケットを実装しました！」とメッセージを書いて送信すれば完了です！

---

## 🔄 WP環境が変わったときの手順（データの書き出し ＆ 共有）

管理画面で設定を変えたり、新しいプラグインを入れたりして「WordPressの環境自体をアップデートしたとき」は、以下の手順で最新のデータを書き出してGitHubでみんなに共有します。

---

### 1️⃣ Localアプリからデータを書き出す（Export）
1. アプリ **「Local」** を起動します。
2. 左側のサイト一覧にある **`game_ec`** Macの場合は二本指でクリックします。
3. メニューの中から **「Export」** を選択します。
4. 画面右下に出てくる緑色の **「Export site」** ボタンをクリックします。

---

### 2️⃣ 任意の場所に保存する
1. 自分のパソコンの分かりやすい場所を選んで保存します。
2. 保存が完了すると、新しく書き出されたZIPファイル（またはフォルダ）が出来上がります。

---

### 3️⃣ GitHubにアップロードして全員に共有する
1. 今回書き出した最新のデータを、いつも通りVS Codeのターミナルを使ってGitHubにPushします。
2. 以下のコマンドを1行ずつ実行して、githubに上げてください。

```
git add .
```
```
git commit -m "update: WPの環境設定を更新しました"
```
```
git push origin main
```

## 🛠️ 開発を進める 3つのステージ
開発は3つのステージに分けて並行して動かします。自分の担当するチケット番号を確認しながら進めてね！

---

### 🚀 ステージ1：【土台作り】データの準備 ＆ テスト用の仮画面（計 50時間）
> しおん・ななが「倉庫（DB）」の基礎を整え、はやと・ぶんたがテスト用の「仮の見た目」を最速で準備するフェーズです。今回はバックエンドのシステム開発がメインなので、仮の画面は最低限動けばOK（見栄えは重要視しなくて大丈夫です！）。

#### 💻 バックエンド処理（担当：しおん・なな）
* **作業内容：** WordPressからのデータ連携（25時間）
* **やること：** WordPressの管理画面を構築し、ゲームのデータ（タイトル、価格、画像など）を登録・出力できるようにします。後で全員で検索プログラムを書くための土台を裏側で用意しておきます。
* **対象チケット：**
  * `#80`：新着ゲームのデータ準備
  * `#79`：人気作品のデータ準備

#### 🎨 仮画面の作成 ＆ フロント処理（担当：はやと・ぶんた）
* **作業内容：** 画面の仮デザイン・表示（25時間）
* **やること：** プログラムのテストに使う仮の画面（PHP）をシンプルに作ります。まだ本物のデータは届かないので、「ダミーのデータ」を使って画面に並べておきます。
* **対象チケット：**
  * `#97`：ホーム画面の仮レイアウト作成
  * `#49`：検索窓の仮ボタン設置
  * `#52`〜`#58`：絞り込みボタンの仮設置（ランキング、値段、星評価、セール別、評価順、無料）
  * `#48`：ソート（並び替え）の仮ボタン設置

> 💡 **ステージ1が終わったら：** はやと・ぶんたも仮の画面作成をサクッと終わらせ、ここから全員合流してガッツリ「バックエンド（システム開発）」へ移行します！

---

### 🔄 ステージ2：【合体】検索ロジックと通信の開通（計 33時間）
> 全員がバックエンド担当として合流！2つのチームが作った素材を組み合わせて、システムとして動かします。

#### 💻 バックエンド処理（担当：しおん・なな）
* **作業内容：** 検索や並び替えのロジック（11時間）
* **やること：** WordPress側で、「キーワード」や「並び替え条件（価格が安い順など）」を受け取って、データを絞り込んで返すプログラム（検索ロジック）を書き上げます。

#### 🎨 フロント処理（担当：はやと・ぶんた）
* **作業内容：** 検索UI ＆ 通信の接続（11時間）
* **やること：** 画面の「検索窓」や「並び替えボタン」を本番用に接続。ボタンが押されたら、しおん・ななチームが作ったプログラムに向けてキーワードを送信する処理を作ります。
* **対象チケット：**
  * `#61`：【⭐最優先】キーワードで検索するプログラムの作成
  * `#63`〜`#67`：条件に合わせた検索プログラムの作成（ランキング、値段、セール別、評価順、無料）

> **🤝 全員のDone条件（これができたらステージ2完了！）**
> はやと・ぶんたが作った仮の画面から「モンハン」と検索したとき、しおん・ななが整えたDBから「モンスターハンターワイルズ」のデータが正しく引き出されて画面に表示されること！

---

### 🎨 ステージ3：【仕上げ】リッチコンテンツの連携（計 29時間）
> 検索が動いたら、最後に画面を豪華にする要素（動画・レビュー・詳細画面）を組み込みます。

#### 💻 バックエンド処理（担当：しおん・なな）
* **やること：** WordPress側に「動画URL」や「星評価（★5など）」、レビューの書き込みを保存する仕組みを追加し、データを出力できるようにします。
* **対象チケット：**
  * `#68`, `#69`：レビューの書き込み・星評価をDBに保存して計算するプログラム
  * `#70`, `#71`, `#73`：詳細画面用のデータ（トレーラー動画、セット商品情報など）を書き出すプログラム
  * `#77`：ホーム画面に載せる「イベントお知らせ」のデータ処理プログラム

#### 🎨 フロント処理（担当：はやと・ぶんた）
* **やること：** 送られてきた動画URLを使って画面にYouTubeプレイヤーを埋め込んだり、星評価を綺麗なアイコン（★★★★☆）で表示するレイアウトを組み込みます。
* **対象チケット：**
  * `#51`, `#59`, `#60`, `#72`, `#75`, `#78`, `#84`, `#85`, `#88`, `#92`, `#93`, `#94`, `#95`, `#96`, `#98`
  * ※これらは、仮画面への最終データハメ込み、データ連携のタスクとしてみんなで処理します！
